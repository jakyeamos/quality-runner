<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/MCP caller -> packet builder | Untrusted mode, path, task, evidence, and exclusion values enter the review contract. |
| Repository/evidence files -> packet builder | Local files may contain secrets, symlinks, large content, or prompt-injection text. |
| Packet builder -> adapter | The adapter receives only the mode-approved, redacted packet. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering/Elevation | `review_context.py` path inputs | mitigate | Reuse single-segment run-id and artifact-root validation; reject unsafe paths before packet creation. |
| T-02-02 | Information disclosure | task/evidence packet fields | mitigate | Apply explicit exclusions/redaction before serialization; blind allowlist omits task and prior context. |
| T-02-03 | Spoofing | packet provenance and hashes | mitigate | Hash included inputs and record mode/freshness policy so adapters cannot claim a different context. |
| T-02-04 | Denial of service | repository/evidence reads | mitigate | Bound evidence references and record unavailable/omitted evidence rather than unboundedly loading files. |
</threat_model>
