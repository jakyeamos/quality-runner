<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Normalized report -> local filesystem | Report content may include task/evidence data and must stay under the run directory. |
| Report -> fixing-agent prompt | Findings may influence a separate agent but must not grant QR mutation authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Information disclosure | report and packet renderers | mitigate | Apply exclusions/redaction before rendering; persist only local artifacts and record unavailable evidence. |
| T-02-06 | Tampering | artifact persistence | mitigate | Reuse safe artifact writers and validate report schema before write. |
| T-02-07 | Spoofing | no-issue/complete summary | mitigate | Emit `review-complete` only after validated adapter result; preserve `review-not-run` separately. |
| T-02-08 | Elevation | fix prompt | mitigate | State that QR does not edit source and fixing agent must investigate and obtain approval within declared scope. |
</threat_model>
