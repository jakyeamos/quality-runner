<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arguments -> review payload | User-controlled task, paths, exclusions, evidence, and adapter options. |
| File adapter -> QR | A returned report may be malformed, oversized, or prompt-injected. |
| QR -> local filesystem | Output must stay within the target repository’s artifact root. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering/Elevation | CLI and adapter paths | mitigate | Validate repo/evidence/output paths, reject traversal/symlinks, and allow writes only through artifact helpers. |
| T-02-10 | Spoofing | adapter result status | mitigate | Validate schema and require explicit adapter completion; no adapter returns `review-not-run`. |
| T-02-11 | Information disclosure | packet and evidence inputs | mitigate | Honor exclusions/redaction and do not invoke remote services from core QR. |
| T-02-12 | Denial of service | returned report size | mitigate | Bound local adapter reads and reject oversized or structurally invalid payloads with a visible limitation. |
</threat_model>
