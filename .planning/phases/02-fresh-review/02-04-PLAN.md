<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project state -> known-issue file | Local state can be edited or stale and must not silently hide findings. |
| Review cycle -> fixing agent | Selected findings become instructions for another agent, not QR mutation commands. |
| Prior cycle -> fresh reviewer | Historical reports are a prohibited input until final summarization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Tampering | known-issues.json | mitigate | Validate schema, preserve edit/remove history, and show all known entries before normal review. |
| T-02-14 | Spoofing | resolution status | mitigate | Require evidence or user confirmation for `resolved`; keep uncertain states explicit. |
| T-02-15 | Information disclosure | active cycle context | mitigate | Pass no prior review artifacts to packet builders until end-of-cycle summarization. |
| T-02-16 | Elevation | fixing-agent handoff | mitigate | Limit prompts to selected findings and state QR read-only boundaries and scope. |
</threat_model>
