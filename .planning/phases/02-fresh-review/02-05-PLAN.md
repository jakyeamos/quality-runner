<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller -> review tool | Agent-provided JSON arguments can request paths, modes, evidence, and adapters. |
| Documentation -> user expectation | Incorrect docs could cause users to expect remote calls or automatic fixes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-17 | Tampering/Elevation | MCP tool arguments | mitigate | Reuse CLI payload validation and explicit MCP schemas; reject unknown fields and unsafe paths. |
| T-02-18 | Spoofing | MCP completion response | mitigate | Return canonical adapter status and artifact paths from the same normalized payload as CLI. |
| T-02-19 | Information disclosure | MCP evidence/packet arguments | mitigate | Keep core local-only, honor exclusions/redaction, and expose permission/access limitations. |
| T-02-20 | Repudiation | docs versus implementation | mitigate | Add release-doc assertions for flags, artifact names, caveat, and safety boundary. |
</threat_model>
