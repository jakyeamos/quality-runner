# v0.3.1 DOI-Ready Release Notes

## Archived In This Release

- Quality Runner source code, CLI, MCP server, and compatibility surfaces.
- Artifact contracts, standards profile docs, agent usage docs, and
  troubleshooting docs.
- Case-study, dogfood, release, threat-model, and example handoff docs.
- Citation, Zenodo, and research-readiness metadata.

## Excluded From The Archive

- Generated `.quality-runner/` outputs from private or unrelated target repos.
- Local run caches, build outputs, and environment files.
- Any target-repo evidence containing secrets or private paths.

## Validation Path

```bash
uv run ruff check quality_runner tests
uv run pytest -q
uv run quality-runner release-smoke --json
```

## Supported Claims

- Quality Runner separates evidence-backed findings from opinionated remediation
  planning.
- `refresh` produces structured scan, standards, capability, audit, remediation,
  ledger, and handoff artifacts.
- Version `0.3.1` includes compatibility surfaces for prior
  `quality_evidence_contract` and `repo_quality_certifier` consumers.

## Required Caveats

- Quality Runner does not edit source files, install target dependencies, create
  commits, call remote services, or execute remediation in version 1.
- Output quality depends on target-repo gates, local environment, and available
  evidence.
- Generated target-repo artifacts are run evidence and should not be cited as
  benchmark results without a defined corpus.
- The format and basedpyright gates are not clean at the time of this
  DOI-readiness pass; clear those blockers before minting an archival DOI.

## DOI Minting Checklist

- Replace the placeholder ORCID in citation metadata.
- Confirm example artifacts contain no private target-repo evidence.
- Run the validation path on the release commit.
- Tag the DOI-ready release as `v0.3.1` or a later explicit release version.
